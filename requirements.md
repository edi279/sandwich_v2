# 샌드위치 레시피 공유 서비스 요구사항정의서

## 1. 프로젝트 개요

### 1.1 프로젝트명
샌드위치 레시피 공유 서비스 (Sandwich Recipe Sharing Service)

### 1.2 프로젝트 목적
샌드위치 레시피와 관련 정보를 공유할 수 있는 커뮤니티 플랫폼 구축

### 1.3 프로젝트 범위
- 레시피 공유 및 조회 기능
- 정보 공유 커뮤니티 기능
- 사용자 관리 및 개인화 기능
- 관리자 기능

## 2. 기능적 요구사항

### 2.1 회원 관리 (Member Management)

#### 2.1.1 로그인 기능
- **FR-001**: 이메일을 통한 로그인
- **FR-002**: 네이버 소셜 로그인 연동
- **FR-003**: 로그인 상태 유지 기능

#### 2.1.2 회원가입 기능
- **FR-004**: 이메일을 통한 회원가입
  - 필수 입력: 닉네임, 이메일, 비밀번호
  - 선택 입력: 이벤트/광고 수신 동의
- **FR-005**: 네이버 연동 회원가입
- **FR-006**: 이메일 중복 검증
- **FR-007**: 닉네임 중복 검증

#### 2.1.3 비밀번호 관리
- **FR-008**: 비밀번호 찾기 기능
  - 이메일 입력 시 임시 비밀번호 발송
- **FR-009**: 비밀번호 변경 기능

### 2.2 홈 화면 (Home Screen)

#### 2.2.1 이슈 토픽 표시
- **FR-010**: 정보공유 카테고리에서 상위 10-15개 태그 표시
- **FR-011**: 태그별 인기도 기반 정렬

#### 2.2.2 추천 레시피 표시
- **FR-012**: 데일리 식단, 소풍/홈카페 카테고리 레시피 추천
- **FR-013**: 추천수 기반 정렬
- **FR-014**: 모바일: 최대 6개, 웹: 최대 10개 표시

### 2.3 레시피 공유 (Recipe Sharing)

#### 2.3.1 카테고리 관리
- **FR-015**: 데일리 식단 카테고리
- **FR-016**: 소풍/홈카페 카테고리

#### 2.3.2 게시글 조회
- **FR-017**: 갤러리 형태 게시글 목록 표시
  - 모바일: 2x6 레이아웃 (페이지당 12개)
  - 웹: 5x5 레이아웃 (페이지당 25개)
- **FR-018**: 페이지네이션 기능
- **FR-019**: 정렬 옵션: 월간 추천수/전체 추천수/최신순

#### 2.3.3 게시글 상세 조회
- **FR-020**: 제목, 재료, 조리과정 순서로 표시
- **FR-021**: 추천 버튼 기능
- **FR-022**: 북마크 기능
- **FR-023**: 댓글 섹션 표시

#### 2.3.4 게시글 작성/수정
- **FR-024**: 레시피 작성 폼
  - 제목, 재료 입력 필드
  - 단계별 입력: 이미지 업로드, 내용, 소스, 팁
- **FR-025**: 단계 추가/삭제 기능
- **FR-026**: 이미지 업로드 기능
- **FR-027**: 게시글 수정 기능 (작성자만)

#### 2.3.5 댓글 기능
- **FR-028**: 댓글 작성 기능
- **FR-029**: 댓글 조회 기능

### 2.4 정보공유 (Information Sharing)

#### 2.4.1 카테고리 관리
- **FR-030**: 재료 처치 카테고리
- **FR-031**: 조리 도구 카테고리
- **FR-032**: 소스 배합 카테고리
- **FR-033**: 기타 카테고리

#### 2.4.2 게시글 조회
- **FR-034**: 리스트 형태 게시글 목록 표시
- **FR-035**: 페이지당 20개 게시글 표시
- **FR-036**: 페이지네이션 기능
- **FR-037**: 정렬 옵션: 주간 추천수/전체 추천수/최신순

#### 2.4.3 게시글 상세 조회
- **FR-038**: 제목, 본문 순서로 표시
- **FR-039**: 추천 버튼 기능
- **FR-040**: 북마크 기능
- **FR-041**: 댓글 섹션 표시

#### 2.4.4 게시글 작성/수정
- **FR-042**: 정보공유 게시글 작성 폼
  - 제목 입력 필드
  - 이미지/텍스트 에디터
- **FR-043**: 게시글 수정 기능 (작성자만)
- **FR-044**: 게시글 삭제 기능 (작성자만)

### 2.5 My 화면 (My Screen)

#### 2.5.1 저장내역
- **FR-045**: 북마크한 게시글 목록 표시
- **FR-046**: 리스트 형태로 표시 (페이지당 20개)
- **FR-047**: 북마크 해제 기능
- **FR-048**: 삭제된 게시글 자동 제거

#### 2.5.2 나의 기록 (배지 시스템)
- **FR-049**: 배지 시스템 구현
  - BA001 (환영해요): 회원 가입 후 최초 게시글 작성
  - BR001 (맛의 시작): "이렇게 만들어요" 게시판 게시글 최초 1회 작성
  - CW001 (교류의 시작): 댓글 최초 5개 작성
  - LG001 (소문의 시작): 내가 작성한 게시글이 최초로 추천 5회를 받음
  - CG001 (이슈의 시작): 내가 작성한 게시글에 댓글이 최초로 10개 이상 달림
  - LW001 (따스한 추천자): 내가 추천한 게시글이 누적 30개
- **FR-050**: 3x4 그리드 형태 배지 표시

##### 뱃지 코드 체계
뱃지 코드는 성취 조건을 간략화하여 관리하기 위해 부여되었습니다:
- **BA**: Board All (모든 게시판 대상)
- **BR**: Board Recipe (이렇게 만들어요-레시피 작성 게시판 대상)
- **CW**: Comment Write (사용자가 댓글을 작성)
- **CG**: Comment Get (사용자의 게시글에 댓글이 달림)
- **LG**: Like Get (사용자가 추천을 받음)
- **LW**: Like Write (사용자가 타인의 글을 추천함)

#### 2.5.3 내 정보 관리
- **FR-051**: 닉네임 변경 기능
- **FR-052**: 이메일 변경 기능
- **FR-053**: 비밀번호 변경 기능

#### 2.5.4 회원 탈퇴
- **FR-054**: 회원 탈퇴 기능
- **FR-055**: 탈퇴 시 게시글은 삭제되지 않음

### 2.6 관리자 기능 (Admin)

#### 2.6.1 사용자 관리
- **FR-056**: 사용자 검색 기능 (닉네임, 이메일)
- **FR-057**: 사용자 정보 조회 (닉네임, 이메일, 광고 수신 동의, 탈퇴 상태, 차단 상태)
- **FR-058**: 사용자 차단 기능
- **FR-059**: 차단된 사용자 게시글 작성 제한

#### 2.6.2 게시글 관리
- **FR-060**: 모든 게시글 삭제 권한
- **FR-061**: 게시글 삭제 확인 모달

### 2.7 기타 기능

#### 2.7.1 Footer
- **FR-062**: 이용약관 링크
- **FR-063**: 개인정보 처리방침 링크
- **FR-064**: 이용안내 링크
- **FR-065**: 연락처 이메일 표시

## 3. 비기능적 요구사항

### 3.1 성능 요구사항
- **NFR-001**: 페이지 로딩 시간 3초 이내
- **NFR-002**: 동시 사용자 1,000명 지원
- **NFR-003**: 이미지 업로드 최대 10MB

### 3.2 보안 요구사항
- **NFR-004**: 사용자 비밀번호 암호화 저장
- **NFR-005**: 세션 관리 및 보안
- **NFR-006**: XSS, CSRF 공격 방지
- **NFR-007**: 파일 업로드 보안 검증

### 3.3 사용성 요구사항
- **NFR-008**: 반응형 웹 디자인 (모바일/웹 지원)
- **NFR-009**: 직관적인 사용자 인터페이스
- **NFR-010**: 접근성 고려 (WCAG 2.1 AA 수준)

### 3.4 호환성 요구사항
- **NFR-011**: Chrome, Firefox, Safari, Edge 브라우저 지원
- **NFR-012**: 모바일 브라우저 지원
- **NFR-013**: 네이버 로그인 API 연동

### 3.5 확장성 요구사항
- **NFR-014**: 향후 소셜 로그인 확장 가능
- **NFR-015**: 카테고리 추가 확장 가능
- **NFR-016**: 다국어 지원 확장 가능

## 4. 시스템 아키텍처 요구사항

### 4.1 기술 스택
- **Frontend**: React.js 또는 Vue.js
- **Backend**: Node.js (Express) 또는 Python (Django/Flask)
- **Database**: PostgreSQL 또는 MySQL
- **File Storage**: AWS S3 또는 로컬 스토리지
- **Authentication**: JWT 토큰 기반 인증

### 4.2 데이터베이스 설계
- **사용자 테이블**: 사용자 정보, 인증 정보
- **게시글 테이블**: 레시피/정보공유 게시글
- **댓글 테이블**: 게시글별 댓글
- **북마크 테이블**: 사용자별 북마크 정보
- **추천 테이블**: 게시글별 추천 정보

## 5. 제약사항

### 5.1 기술적 제약사항
- 구글 로그인 API 의존성
- 이미지 파일 형식 제한 (JPG, PNG, GIF)
- 파일 크기 제한

### 5.2 비즈니스 제약사항
- 사용자 생성 게시글은 탈퇴 시에도 유지
- 차단된 사용자는 게시글 작성 불가
- 관리자만 모든 게시글 삭제 가능

## 6. 우선순위

### 6.1 높은 우선순위 (MVP)
- 게시판 기본 기능
- 게시글 조회/작성 기능
- 기본적인 사용자 관리

### 6.2 중간 우선순위
- 회원가입/로그인 기능
- 정보공유 기능
- 북마크 기능
- 댓글 기능
- 배지 시스템

### 6.3 낮은 우선순위
- 관리자 기능
- 고급 검색 기능
- 통계 기능

## 7. 성공 기준

### 7.1 기능적 성공 기준
- 모든 핵심 기능이 정상 작동
- 사용자 시나리오 테스트 통과
- 관리자 기능 정상 작동

### 7.2 비기능적 성공 기준
- 페이지 로딩 시간 3초 이내 달성
- 동시 사용자 1,000명 지원
- 보안 취약점 없음
- 크로스 브라우저 호환성 확보
